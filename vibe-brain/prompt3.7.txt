Main requirements:

1. Global app navigation
Improve the app shell navigation so users always know where they are.

Add or refine:
- Logo/title link to /
- “Command Center” or “Apps” link to /apps
- Current app/page active state
- ThemeSwitcher visible in header
- Responsive mobile nav if practical
- Safe back navigation

Navigation labels should be in Bahasa Indonesia where natural:
- Beranda
- Command Center
- Apps
- Kembali ke Apps
- Berikutnya
- Sebelumnya

2. Consistent app page header
Each app page should have a consistent header area.

Every app route should show:
- app title
- category
- short description
- status/risk badge
- AI action label or AI-ready indicator
- button/link: “Kembali ke Command Center”
- optional next/previous app navigation

Use app metadata as the single source of truth where practical.

Do not manually duplicate route/title/category mappings across every page if metadata already exists.

3. App directory enhancement
Improve /apps as the main app directory.

The /apps page should include:
- title: Command Center Apps
- subtitle: Pilih mini-app untuk menjalankan workflow security: Detect, Triage, Patch, Verify, Deploy.
- all 10 app cards
- visible category
- visible priority
- visible status/risk badge
- visible AI action name
- route link
- hover glow
- selected theme styling
- responsive grid

Add lightweight filtering if practical:
- by category
- by priority
- by workflow phase

Do not overbuild.

4. Cross-app demo flow
Add a simple demo flow indicator that helps judges/users understand how the apps connect.

The flow should show:
Detect → Triage → Patch → Verify → Deploy

Each app should optionally map to one or more workflow phases.

Example mapping:
- Threat Runner: Detect
- Risk Dashboard: Triage
- CVSS Risk Calculator: Triage
- Patch Monitor: Patch
- Patch Deadline Countdown: Patch
- Incident Response Journey: Verify / Deploy / Postmortem
- CVE Resource Library: Education / Support
- Threat Briefing Player: Detect / Education
- SecureOps Habit Tracker: Monitoring / Hygiene
- Analyst Portfolio Builder: Portfolio / Career

Use these mappings in the /apps page or app cards if practical.

5. Next/previous app navigation
Add simple next/previous navigation between app pages if practical.

For each app page:
- “Sebelumnya” links to previous app route
- “Berikutnya” links to next app route
- “Kembali ke Command Center” links to /apps

This must not cause 404.
Use centralized metadata order.

1. Mulai dari Threat Runner untuk simulasi deteksi threat.

All navigation and directory improvements must use the existing global Theme Engine.

Requirements:

- Do not create a second ThemeSwitcher.
- Do not create page-specific theme toggles.
- App cards, nav active states, badges, hover glows, demo guide panels, and workflow indicators must follow the selected theme.
- ThemeSwitcher must still work.
- Selected theme must persist after refresh.

9. Gemini AI safety
Do not change Gemini integration unless required for broken imports.

Requirements:
- GEMINI_API_KEY stays server-side only.
- No client-side Gemini calls.
- No PUBLIC_ or NEXT_PUBLIC_ Gemini key.
- No raw Gemini errors exposed.
- All AI output remains Bahasa Indonesia.
- Defensive security only.
- Do not generate offensive exploit instructions.
