Important implementation note:

Suggested internal route:
- src/routes/api/security-headers/+server.ts

Expected request:
{
  url: string
}

Expected response:
{
  ok: boolean,
  url: string,
  grade: "A" | "B" | "C" | "D",
  score: number,
  presentHeaders: string[],
  missingHeaders: string[],
  checkedHeaders: array,
  summary: string,
  error?: string
}

Headers to check:
- Content-Security-Policy
- Strict-Transport-Security
- X-Content-Type-Options
- X-Frame-Options
- Referrer-Policy
- Permissions-Policy
- Cross-Origin-Opener-Policy
- Cross-Origin-Resource-Policy
- Cross-Origin-Embedder-Policy

Grade suggestion:
- A: score >= 85
- B: score >= 70
- C: score >= 50
- D: score < 50

Scoring:
- Weight important headers higher:
  - Content-Security-Policy
  - Strict-Transport-Security
  - X-Content-Type-Options
  - X-Frame-Options
  - Referrer-Policy
  - Permissions-Policy
- Other headers can add smaller points.
- Keep scoring simple and transparent.

If real server-side fetch fails:
- show friendly Indonesian error
- do not crash
- do not expose stack trace
- allow fallback demo sample result if practical

UI requirements:
- Header input card

- Header recommendation list

- Loading scanner state
- Error state
- Empty state with sample domain suggestion

  checkedHeaders: array
}
