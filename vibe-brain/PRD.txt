# VibeSec Lab by Andy — Product Requirements Document

## 1. Ringkasan Produk

VibeSec Lab by Andy adalah premium cyber command center playground untuk CVE intelligence, security header hardening, patch planning, incident response, defensive AI workflows, security habits, dan security portfolio building.

Produk ini dibuat untuk demo vibe coding yang matang: visualnya cinematic, scope-nya jelas, AI-nya defensif, navigasinya route-safe, dan deploy-nya siap Cloud Run.

## 2. Visi dan Narasi Produk

VibeSec Lab harus terasa seperti command center security modern: dark glass UI, cyber bento grid, terminal-inspired, premium, readable, dan demo-ready.

### Security Flow

```txt
Detect → Triage → Patch → Verify → Deploy
```

### Story Produksi

```txt
Design → Security PRD + system scope
Build  → Antigravity + Gemini + SvelteKit
Launch → Cloud Run
```

### Pesan Utama

VibeSec Lab menunjukkan perbedaan antara:

- **Raw vibe coding:** cepat, tapi scope mudah drift.
- **Mature vibe coding:** pakai PRD, design system, route map, AI contracts, QA, dan deployment plan.

## 3. Tujuan dan Non-Tujuan

### Tujuan

- Membangun satu SvelteKit app berisi 10 mini-app security-themed.
- Menghadirkan cyber command center yang premium dan modern.
- Menyediakan global ThemeSwitcher dengan persistensi tema.
- Mengintegrasikan Gemini AI server-side untuk seluruh app.
- Menjaga output AI dalam Bahasa Indonesia.
- Fokus AI pada defensive security.
- Menjaga app demo-ready, Cloud Run-ready, dan route-safe.

### Non-Tujuan

- Bukan SIEM sungguhan.
- Bukan vulnerability scanner produksi.
- Bukan exploit generator.
- Bukan offensive hacking tool.
- Bukan platform auth/RBAC/database enterprise penuh.
- Bukan multi-tenant SaaS.

## 4. Audiens

- Reviewer demo / juri vibe coding.
- Recruiter / client yang melihat portfolio Andy.
- Developer yang tertarik pada AI-assisted security tooling.
- Audience cloud/security yang ingin melihat workflow defensif dalam bentuk interaktif.

## 5. Stack dan Arsitektur

### Stack

- SvelteKit
- Svelte 5
- Tailwind CSS
- Magic UI-style motion polish
- Gemini API via server-side SvelteKit routes
- localStorage untuk persistence
- Web Audio API untuk Threat Runner local sound effects
- SvelteKit adapter-node
- Google Cloud Run target

### Runtime AI Flow

```txt
Svelte UI → /api/ai/security → Gemini server helper → Gemini API
```

### Server-only AI Rules

- Gemini hanya dipanggil dari server-side routes.
- `GEMINI_API_KEY` dibaca dari private server env.
- Tidak boleh ada `PUBLIC_` atau `NEXT_PUBLIC_` Gemini key.
- `/api/ai/health` hanya boleh menampilkan keberadaan key, panjang key, dan nama model.
- Tidak pernah menampilkan nilai key.
- Error provider mentah tidak boleh bocor ke UI.

## 12. AI Architecture dan Safety

### Architecture

```txt
Svelte UI → /api/ai/security → Gemini server helper → Gemini API
```

### Health Endpoint

`/api/ai/health` dipakai untuk diagnostics aman.

Health endpoint boleh menampilkan:

- `hasGeminiKey`
- `geminiKeyLength`
- nama model

Health endpoint tidak boleh menampilkan:

- nilai API key
- stack trace
- secret

### Error Handling

- Jika key hilang / invalid / expired, tampilkan error Bahasa Indonesia yang ramah.
- Raw Gemini/provider errors tidak boleh diekspos.
- Response harus tetap structured.

### AI Safety Principles

#### Allowed

- mitigation
- patching
- hardening
- monitoring
- incident response
- education
- stakeholder updates
- postmortem templates
- defensive portfolio/CV content

#### Disallowed

- exploit instructions
- malware guidance
- payload generation
- credential theft
- bypass / evasion guidance
- persistence guidance
- weaponization
- offensive playbooks

### Global AI Instruction

```txt
Jawab selalu dalam Bahasa Indonesia yang jelas, ringkas, dan praktis. Fokus pada defensive security: mitigasi, patching, hardening, monitoring, incident response, dan edukasi. Jangan memberikan instruksi eksploitasi ofensif, malware, bypass, credential theft, atau penyalahgunaan sistem.
```

## 14. Mini-App Requirements

### 14.1 Threat Runner

Category: Retro Arcade Games

Scope:

- cyber defense arcade shooter
- visible robot / defense bot di bagian bawah
- player bergerak kiri/kanan
- player menembak threat dari bawah ke atas
- threat turun dari atas
- player mengumpulkan stars / patch tokens untuk score
- health, score, timer, threat level, high score
- local sound effects
- sound toggle
- sound hanya mulai setelah user interaction
- Web Audio API / local browser audio only
- tidak ada external audio API

### 14.2 Risk Dashboard

Route: `/apps/risk-dashboard`

Konsep: **Security Header Scanner**

Scope:

- user memasukkan domain atau URL
- server-side security header scan
- checks headers:
  - Content-Security-Policy
  - Strict-Transport-Security
  - X-Content-Type-Options
  - X-Frame-Options
  - Referrer-Policy
  - Permissions-Policy
  - Cross-Origin-Opener-Policy
  - Cross-Origin-Resource-Policy
  - Cross-Origin-Embedder-Policy
- grade A/B/C/D
- score
- present headers
- missing headers
- Gemini hardening suggestions dalam Bahasa Indonesia
- defensive only

### 14.3 CVSS Risk Calculator

Scope:

- menghitung priority risk secara contextual
- menampilkan severity, asset criticality, exposure, exploit availability, data sensitivity, patch complexity
- output risk score + severity + mitigation explanation
