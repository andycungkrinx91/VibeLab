- Cyber arcade style

- Not loud
- Not annoying
- Theme-friendly feel

Audio controls:

- toggle: sound on/off
- label in Bahasa Indonesia:
  - “Suara”
  - states: “Aktif” / “Mati”
- optional volume slider if easy and safe
- selected sound setting should persist in localStorage if practical

Browser/audio safety:
- Audio must initialize only after user interaction.
- No autoplay on page load.
- No console errors if audio is blocked.
- If Web Audio is unavailable, fail gracefully.
- Do not crash SSR.
- Guard window/AudioContext access safely.
- Clean up audio resources if needed.

Suggested implementation:
- Create a small local audio utility if useful:
  - src/lib/audio/game-audio.ts
- Use Web Audio API oscillator tones for simple generated effects.
- Export functions such as:
  - initAudio()
  - playStart()
  - playShoot()
  - playHit()
  - playCollect()
  - playDamage()
  - playGameOver()
  - setSoundEnabled()
- Keep utility browser-safe.
- Do not import browser-only audio code into server files.

Threat Runner gameplay integration:

Wrap audio calls safely.
